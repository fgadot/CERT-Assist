Changelog

CERT Command — Release history and updates

v1.4
July 21, 2026
Update
CERT Activation
🆕Team leaders now explicitly activate their team with an ACTIVATE CERT button at the top of the dashboard
🆕County EOC only sees teams that have been explicitly activated — no longer based on member check-in count
🆕Activation state persists through dashboard restarts and server reboots — remains active until team leader deactivates
🆕Both activation and deactivation require a confirmation prompt to prevent accidental changes
🆕Team leader can deactivate at the end of an incident to remove the team from the county view
🆕A team flag sent to county is now possible even when no members are checked in, as long as the team is activated
Sub-Team Assignment
🆕Replaced text-entry prompt for adding a member to a sub-team with a proper checkbox modal
🆕New → Team button on each member card lets team leader assign a member to a sub-team directly from the roster
ICS-213 Export
🆕Any incident report can now be exported as a FEMA ICS-213 General Message form — printable from both the team dashboard and county EOC
🆕Form auto-fills To, From, Subject, Date/Time, Message, Reply, and Replied By from existing report data
v1.3
July 21, 2026
Update
Live Member Map
🆕Interactive map panel on the team leader dashboard shows all checked-in members with GPS pins
🆕Pins are color-coded by member status (green = Available, blue = On Task, orange = Need Help, red = Injured)
🆕Tap the 📍 button next to any member in the roster to zoom the map directly to their location
🆕Map auto-fits to show all members when new location data arrives
🆕Location timestamp shown in roster ("📍 2 minutes ago") so team leader knows how fresh each position is
Remote Check-Out Notification
🆕Members are now notified within 10 seconds when a team leader checks them out from the dashboard
🆕iPhone app displays an alert: "Your Team Leader has checked you out" and returns to check-in screen automatically
🆕Web member portal detects the same event and shows the message, then clears the session
🆕If the app was in the background or killed, the notification is shown on next launch
Version Enforcement
🆕Server now publishes a minimum required app version — outdated clients are blocked at check-in
🆕App displays a full-screen "Update Required" message when the server requires a newer version
🆕Version requirement is configurable via config/versions.json without redeploying
Fixes
🔧Server address field now defaults to http:// — local IP addresses work without typing a prefix
🔧Explicit https:// addresses still pass through unchanged (production unaffected)
v1.2
July 2026
Update
Incident Log & FEMA Compliance
🆕All significant events now logged to a persistent database (survives server restarts)
🆕Incident Log panel on the dashboard shows every check-in, status change, report, task, and reply
🆕Export to ICS-214 Activity Log — FEMA/NIMS-compliant printable PDF via browser
🆕ICS-211 Check-In/Check-Out roster embedded in the export (names, roles, equipment, times)
🆕Events captured: member check-in/out, status changes, report submissions, report replies, task creation/completion, sub-team creation

Initial Release
Member Portal
✅Browser-based check-in — no app required
✅Member PIN separate from team leader dashboard PIN
✅ICS role selection at check-in
✅Real-time assignment and task display (auto-refreshes every 10 s)
✅ICS-213 incident report submission
✅Reply inbox — team leader responses visible in-app
Team Leader Dashboard
✅Live member roster with status and role
✅Sub-team creation and member assignment
✅Task board — create, assign, edit, and cancel tasks
✅Incoming report queue with reply capability
✅Escalation to County EOC with one click
✅Member name editing directly from roster
✅WebSocket-based live updates (no polling)
County EOC Dashboard
✅Unified view of all 5 team feeds
✅Escalated reports from any team land here automatically
✅County reply routed back to originating team
Infrastructure & Security
🔒HTTPS on all 6 subdomains (wildcard cert, TLS 1.2/1.3)
🔒fail2ban — IP blocked after 2 wrong PIN attempts for 1 hour
🔒Rate limiting — 60 req/min per IP on all API endpoints
🔒Docker containers bound to localhost only (ports not exposed to internet)
🔒Security headers: X-Frame-Options, CSP, X-Content-Type-Options
🔒UFW firewall — only ports 22 (restricted), 80, and 443 open
